## 10. Priority Levels

Support requests may be classified by priority.

Priority may be adjusted as more information becomes available.

### Priority 1: Critical

A Priority 1 issue may include:

- Widespread service outage
- Critical production impact
- Confirmed security incident
- Confirmed data exposure
- Inability for an enterprise customer to access core services
- Failed authentication affecting many users
- Dangerous automation behavior
- Active exploitation of a Helix AI vulnerability
- Major billing system failure affecting many customers
- Severe data integrity issue

### Priority 2: High

A Priority 2 issue may include:

- Major feature unavailable
- Significant service degradation
- Important integration failure
- Important API failure
- Organization administration failure
- Workspace access issue affecting multiple users
- High-impact automation failure
- High-impact marketplace or plugin issue
- Security concern requiring investigation
- Billing issue blocking paid access

### Priority 3: Medium

A Priority 3 issue may include:

- Non-critical product bug
- Isolated feature issue
- Limited integration issue
- Limited automation issue
- Dashboard or analytics issue
- Documentation issue that affects usage
- API issue with available workaround
- Billing question not blocking service
- Marketplace issue not creating immediate risk

### Priority 4: Low

A Priority 4 issue may include:

- General question
- Feature request
- Documentation suggestion
- Minor user interface issue
- Cosmetic issue
- Non-urgent configuration question
- Product feedback
- Enhancement request
